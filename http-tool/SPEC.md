# HTTP Tool 规格说明书

## 1. 项目概述

- **项目名称**: http-tool
- **项目类型**: CLI HTTP 请求工具
- **核心功能**: 通过命令行发送 HTTP 请求并展示格式化响应
- **目标用户**: 开发者、运维人员、测试工程师

## 2. 技术选型

| 项目 | 选择 |
|------|------|
| HTTP 库 | urllib（标准库）|
| 依赖 | 零外部依赖（Python 3 标准库）|
| 同步方式 | 原生同步 |

**选型理由**: Coordinator 指定 urllib，零外部依赖，满足轻量级 CLI 工具需求。

## 3. 功能规格

### 3.1 支持的 HTTP 方法

| 方法 | 支持 | 说明 |
|------|------|------|
| GET | ✅ | 请求指定资源 |
| POST | ✅ | 提交数据 |
| PUT | ✅ | 更新资源 |
| DELETE | ✅ | 删除资源 |

### 3.2 命令行接口

```bash
python http_tool.py <method> <url> [options]
```

### 3.3 选项参数

| 参数 | 格式 | 说明 |
|------|------|------|
| `-H` | `-H header: value` | 添加 HTTP 请求头，可重复使用 |
| `-d` | `-d body` | 设置请求体（body） |

### 3.4 功能详情

1. **请求发送**
   - 使用 `urllib.request.Request` 构造请求
   - 使用 `urllib.request.urlopen` 发送请求
   - 支持所有标准 HTTP 方法

2. **请求头**
   - 通过 `-H` 选项添加
   - 格式: `Header-Name: value`
   - 可添加多个 `-H` 选项

3. **请求体**
   - 通过 `-d` 选项设置
   - 适用于 POST/PUT/DELETE 方法

4. **响应处理**
   - 自动检测 JSON 响应并格式化输出
   - 非 JSON 响应直接输出原始内容
   - 显示 HTTP 状态码

## 4. 数据流

```
用户输入 → 参数解析 → 构建 Request → urlopen → 读取响应 → 格式化输出
```

## 5. 错误处理

| 场景 | 处理 |
|------|------|
| URL 无效 | 打印错误信息并退出 |
| 网络错误 | 打印错误信息并退出 |
| 超时 | 打印错误信息并退出 |
| 非 2xx 响应 | 正常输出响应内容 |

## 6. 限制事项

- 不支持 HTTPS 证书验证（Python 默认行为）
- 不支持请求超时设置（使用系统默认）
- 不支持文件上传
- 不支持重定向控制
